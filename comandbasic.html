  1. git add

    2. Moves changes from the working directory to the staging area. This way you can prepare a snapshot before committing to the official history.

  2. git branch

    3. This command is your general-purpose branch management tool. It allows you to create isolated development environments within a single repository.

  3. Git Checkout

    4. Besides checking out commits and old file revisions, git checkout is also used to navigate through existing branches. Combined with basic Git commands, it is a way to work on a specific development line.

  4. Git Clean

    5. Removes untracked files from your working directory. It is the logical counterpart to git reset, which normally only works on tracked files.

  5. Git Clone

    6. Creates a copy of an existing Git repository. Cloning is the most common way for developers to obtain a working copy of a central repository.

  6. Git commit

    7. Commits the staged snapshot to the project history. Combined with git add, it defines the basic workflow for all Git users.

  7. Git merge

    8. It is an effective way to integrate changes from divergent branches. After branching the project history with git branch, git merge allows you to combine it back.

  8. Git pull

    9. This command is the automated version of git fetch. It downloads a branch from a remote repository and immediately merges it into the current branch. This is Git's equivalent of svn update.

  9. Git push

    10. Pushing is the opposite of fetching, with some exceptions. It allows you to move one or more branches to another repository, which is a good way to publish contributions. It's like svn commit, but sends a series of commits rather than a single set of changes.

  10. Git log

    11. Allows you to explore previous revisions of a project. It provides various formatting options to display the committed snapshots.
